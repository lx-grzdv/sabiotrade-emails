# В разработке

Здесь ведётся вёрстка и правки писем. В каждой подпапке — кампания: `.docx` с текстом, черновики `.html`.

## Workflow

1. Работаете в `development/<кампания>/`
2. Когда письмо готово к показу — копируете HTML в `preview/<кампания>/`
3. Для передачи менеджеру/CRM используете публичный source в `sources/<кампания>/` (генерируется скриптом)
4. Обновляете `index.html` (статус -> «готово», ссылки на превью)
5. Деплой: `npm run deploy` (синхронизирует `preview/` и `sources/`)

## Быстрая синхронизация одной кампании

```bash
npm run sync-preview
npm run sync-sources
```
